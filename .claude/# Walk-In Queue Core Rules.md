# Walk-In Queue Core Rules

This project ONLY builds a live walk-in queue system.

PRIMARY GOAL:
Sync all barbers schedules into a real-time availability queue.

Core Rules:

1. Queue reflects TRUE availability.
2. No double booking allowed.
3. Barber calendar is source of truth.
4. Queue auto updates when schedule changes.
5. Customers see live estimated wait time.
6. Queue must update in real-time (no refresh needed).

Out of Scope:

- payments
- subscriptions
- CRM
- portfolios
- marketing
- tipping
